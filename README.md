# 💬 Discodeit (Discord Clone Project)

**Discodeit**은 커뮤니케이션 플랫폼 '디스코드(Discord)'의 핵심 기능을 모티프로 하여, 대규모 유저와 채널 간의 상호작용을 관리하는 **메시징 서비스 백엔드 시스템**입니다. 

스프린트 3까지의 과정을 통해 객체지향적 설계와 스프링 프레임워크의 핵심 기능을 결합하여 안정적인 비즈니스 로직을 구축하였습니다.

---

## 🚀 핵심 기능 (Core Features)

### 1. 유저 및 커뮤니티 관리
* **실시간 상태 관리**: 마지막 접속 시간을 추적하여 유저의 온라인/오프라인 상태를 실시간으로 판단합니다.
* **프로필 및 바이너리 데이터**: 유저 프로필 이미지와 메시지 첨부파일 등 바이너리 데이터를 체계적으로 관리합니다.
* **데이터 무결성 유지**: 유저나 채널 삭제 시, 이와 연관된 메시지 및 수신 상태(ReadStatus)를 자동으로 정리하여 데이터 정합성을 유지합니다.

### 2. 채널 및 메시징 시스템
* **공개/비공개 채널**: 채널 성격에 따른 접근 제어 로직을 포함합니다.
* **메시지 수신 추적**: 유저별로 채널 메시지를 어디까지 읽었는지 관리하여 "읽지 않은 메시지" 기능을 지원합니다.

---

## 🛠 기술적 특징 (Technical Implementation)

### 🏗 Spring 기반의 유연한 아키텍처
* **IoC/DI 적용**: 스프링 컨테이너를 통해 객체 간 결합도를 낮추고 유지보수성을 극대화했습니다.
* **다중 저장소 전략 (JCF & File)**: 메모리 기반 저장 방식(JCF)과 영속성 저장 방식(File)을 `application.yaml` 설정 하나로 자유롭게 교체할 수 있도록 설계했습니다.
* **Lombok 최적화**: 보일러플레이트 코드를 제거하여 비즈니스 로직의 가독성을 높였습니다.

### 📊 데이터 모델링
* **Time Standard**: 전 세계 유저를 고려하여 모든 시간 데이터를 `Instant` 타입으로 표준화했습니다.
* **DTO 패턴**: 계층 간 데이터 이동 시 DTO(Data Transfer Object)를 사용하여 내부 도메인 모델을 보호하고 필요한 데이터만 노출합니다.
